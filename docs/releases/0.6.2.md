# swift-lm 0.6.2

`0.6.2` is a focused performance patch for the Qwen3.5 DeltaNet path. The
SSM recurrence kernel — used by every DeltaNet layer during both prefill
and decode — now partitions work across GPU cores by key-group and uses
an algebraic identity to eliminate a full state read/write pass per step.

## What 0.6.2 Changes

### SSM recurrence parallelization (Qwen3.5 DeltaNet)

- `MetalDispatchDimension.partitionedReduction(partitionCount:,
  threadsPerPartition:)` is a new dispatch kind for fragments whose
  natural parallelism is "N independent threadgroups, each reducing
  over its own partition". `DispatchHeuristics` maps it to
  `grid=(partitionCount, 1, 1)` with threadgroup size clamped to the
  pipeline's `maxTotalThreadsPerThreadgroup`.
- `SSMRecurrenceFragment.dispatchDimension` now returns
  `partitionedReduction` sized from the fragment's own dimensions. Both
  the prefill (`generateSSMRecurrenceSequence`) and decode
  (`generateSSMRecurrence`) kernels previously dispatched as
  `grid=(1,1,1)`, running on a single GPU core.
- Each threadgroup owns exactly one key-group's Q/K/V conv channels
  and the recurrent-state slice of its heads — disjoint partitions,
  no cross-TG synchronization.

### SSM inner-loop state recurrence

Both kernels now use the algebraic identity

```
state_after[j] = state_before[j]·decay + K[j]·kInvDelta
dot = Σ state_after[j]·Q[j]
    = Σ (state_before[j]·decay)·Q[j] + kInvDelta · Σ K[j]·Q[j]
    = sqSum + kInvDelta · kqSum
```

where `kqSum = Σ Q[j]·K[j]` is precomputed once per head and `sqSum`
accumulates during the decay pass. This eliminates the second Σ over
updated state. Per `j`: **2 state reads + 1 state write** (was 3 reads
+ 2 writes).

## Benchmarks (Qwen3.5-0.8B BF16, 3-run median)

| Metric | 0.6.1 | 0.6.2 | Delta |
|---|---|---|---|
| prefill @ 16 tokens | 335.7 | 658.6 tok/s | +96% |
| prefill @ 32 tokens | 402.7 | 975.0 tok/s | +142% |
| prefill @ 64 tokens | 465.2 (est) | 1284.9 tok/s | — |
| prefill @ 128 tokens | 465.2 | 1414.1 tok/s | +204% |
| decode (100 steps) | 83.7 | 99.7 tok/s | +19% |

3-run coefficient of variation: prefill < 1%, decode < 0.3%. The gains
are genuine (the probe confirmed SSM was 76–92% of prefill time at
seqLen 16–128 on 0.6.1, and decode dispatch serialization on a single
threadgroup was the dominant remaining latency per token).

## Scope

Models currently affected:

- **Qwen3.5** — the only model that uses `DeltaNet()` → `StateSpaceAttributes`
  → `SSMRecurrenceFragment` in its declaration.

Models not affected by this patch:

- **LFM2** uses `ShortConv()` → `Conv1dFragment`, not SSM recurrence.
- **Gemma4, EmbeddingGemma, Llama-family, and other attention
  transformers** do not traverse this code path.

If a future model declaration adds DeltaNet / Mamba / Griffin / RecurrentGemma
patterns on top of `SSMRecurrenceFragment`, it will pick up these gains
automatically.

## Correctness

- `QwenVisionRealBundleTextTests` — both real-bundle text generation
  tests (simple prompt + chat with thinking content) pass unchanged.
- The benchmark exercises 100 decode steps per run across 3 runs with
  no crash, NaN cascade, or throughput regression.
