# swift-lm 0.8.2

`0.8.2` is a thinking-output correctness release. It fixes prompt-opened
reasoning spans in chat templates such as Qwen and LFM, and adds focused E2E
coverage for supported thinking-template families.

## What 0.8.2 Changes

### Prompt-opened reasoning state

Some chat templates open a reasoning span in the rendered prompt before token
generation starts. For example, Qwen templates can end the generation prompt
inside `<think>`. `swift-lm` now detects whether the prompt tail is already
inside the active reasoning tag policy and starts generation visibility state in
that mode.

This means:

- `.separate` emits generated reasoning as `GenerationEvent.reasoning`.
- `.hidden` suppresses prompt-opened reasoning from visible text.
- `.inline` remains unchanged and leaves template-defined reasoning inline.

### Separate reasoning raw-token cap

When `GenerationParameters.reasoning == .separate`, reasoning is surfaced as
output. The raw-token cap now matches the requested visible token limit in this
mode, avoiding excessive hidden-token expansion intended only for `.hidden`.

### E2E thinking coverage

New focused tests cover public `generate(ModelInput, parameters:)` behavior for
thinking-capable template families:

- Qwen `<think>` prompt-opened reasoning
- LFM `keep_past_thinking` / `<think>` semantics
- Gemma channel-tag safety for `<|channel>thought` / `<channel|>`

## Correctness

Validated before release:

```bash
perl -e 'alarm shift; exec @ARGV' 120 swift test --filter GenerationThinkingOptionsTests
perl -e 'alarm shift; exec @ARGV' 120 swift test --filter QwenVisionRealBundleTextTests/realBundlePublicGenerateThinkingSeparateE2E
perl -e 'alarm shift; exec @ARGV' 120 swift test --filter LFMVisibilityBenchmarkDiagnosticsTests/publicGenerateE2EEmitsReasoningStream
perl -e 'alarm shift; exec @ARGV' 120 swift test --filter Gemma4RuntimeTests/realGemma4PublicGenerateThinkingSeparateE2E
perl -e 'alarm shift; exec @ARGV' 120 swift test --filter TemplateThinkingTagPolicyExtractorTests
perl -e 'alarm shift; exec @ARGV' 120 swift test --filter TemplateThinkingStrictnessTests
```

Observed behavior:

- Qwen E2E emits reasoning events for prompt-opened `<think>` spans.
- LFM E2E emits reasoning events for `keep_past_thinking` semantics.
- Gemma4 E2E does not leak channel tags into visible or reasoning output.

## Notes

- No public `SwiftLM` API migration is required.
- Large real-model suites remain best run as focused invocations. The new LFM
  E2E passes as a focused test; the broader diagnostic suite can exceed a
  120-second outer timeout on local machines.

dependencies: [
